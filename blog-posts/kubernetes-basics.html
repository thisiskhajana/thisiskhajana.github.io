Kubernetes Basics

June 2026 Khajana Raju 8 min read

Kubernetes has become the industry standard for container orchestration. It automates the deployment, scaling, and management of containerized applications. If you're working with containers, understanding Kubernetes is crucial for modern DevOps practices.

What is Kubernetes?

Kubernetes (often abbreviated as K8s) is an open-source container orchestration platform. It automates many of the manual processes involved in deploying, managing, and scaling containerized applications across a cluster of machines.

Why Kubernetes?

  • Automated Deployment: Deploy containers across your cluster efficiently
  • Self-Healing: Automatically restart or replace containers that fail
  • Load Balancing: Distribute traffic across containers
  • Automatic Scaling: Scale applications based on demand
  • Rolling Updates: Update applications without downtime
  • Resource Management: Optimize resource allocation

Kubernetes Architecture

Control Plane (Master)

Manages the Kubernetes cluster and makes decisions about the cluster. Components include:

  • API Server: Exposes the Kubernetes API
  • Scheduler: Assigns pods to nodes
  • Controller Manager: Runs controller processes
  • etcd: Key-value store for cluster data

Worker Nodes

Run containerized applications. Components include:

  • Kubelet: Ensures containers run in a pod
  • Container Runtime: Runs containers (Docker, containerd)
  • kube-proxy: Network proxy for communication

Key Concepts

Pod

The smallest deployable unit in Kubernetes. Usually contains one container, but can contain multiple tightly-coupled containers.

Deployment

Describes desired state for pods and replicas. Manages rolling updates and rollbacks.

Service

Exposes pods to network traffic and other pods. Provides load balancing and service discovery.

Namespace

Virtual cluster for resource isolation. Helps organize and manage multi-tenant environments.

ConfigMap & Secret

Store configuration data and sensitive information separately from container images.

Basic Kubernetes Commands

  • kubectl create deployment myapp --image=myimage - Create a deployment
  • kubectl expose deployment myapp --port=8080 - Expose a deployment
  • kubectl scale deployment myapp --replicas=3 - Scale replicas
  • kubectl get pods - List all pods
  • kubectl logs pod-name - View pod logs

Kubernetes Benefits

  • Container orchestration at scale
  • High availability and fault tolerance
  • Easy deployment and rollback
  • Resource optimization
  • Microservices support
  • Active community and ecosystem

Challenges

  • Steep learning curve
  • Complex configuration
  • Resource overhead
  • Operational complexity
  • Requires expertise to optimize

Conclusion

Kubernetes is a powerful tool for managing containerized applications at scale. While it has a learning curve, mastering Kubernetes is essential for modern DevOps engineers. Start with the basics, practice with small clusters, and gradually explore advanced features like Helm, operators, and custom resource definitions.
